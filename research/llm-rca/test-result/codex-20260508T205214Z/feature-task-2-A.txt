**Incident Report**

Most likely root cause: a deliberate server/client render branch in `app/[locale]/(auth)/sign-in/components/LoginComponent.tsx`.

At line 36, the component renders:

```tsx
const cardTitle = typeof window === "undefined" ? "Login" : "Sign in";
```

Because this is a Next.js client component, it is still server-rendered first, then hydrated in the browser. The server therefore emits `Login`, while the browser’s first render emits `Sign in`. That exactly matches the incident payload:

```diff
+ Sign in
- Login
```

There is a second mismatch in the same component:

```tsx
const dividerLabel =
  typeof window === "undefined"
    ? "Or continue with email"
    : "Or continue with Email";
```

So even if the title mismatch were absent, the divider text could also trigger hydration drift.

The regression was introduced by commit `9be250f3` titled `Polish sign-in copy and labels` on 2026-05-01. The incident occurred on 2026-05-04, after that change.

**Conclusion**

This is not caused by i18n, auth state, locale loading, or browser extensions. The root cause is render-time use of `typeof window` to choose visible text, producing different initial HTML on the server and client.

**Recommended fix**

Make the initial render deterministic: use the same literal text on both server and client, for example always `Sign in` or always `Login`. If browser-only behavior is genuinely needed, defer it until after hydration with state updated from `useEffect`, but visible static copy should not depend on `typeof window`.

**Incident Report**

Most likely root cause: a deliberate server/client render branch in `app/[locale]/(auth)/sign-in/components/LoginComponent.tsx`.

At line 36, the component renders:

```tsx
const cardTitle = typeof window === "undefined" ? "Login" : "Sign in";
```

Because this is a Next.js client component, it is still server-rendered first, then hydrated in the browser. The server therefore emits `Login`, while the browser’s first render emits `Sign in`. That exactly matches the incident payload:

```diff
+ Sign in
- Login
```

There is a second mismatch in the same component:

```tsx
const dividerLabel =
  typeof window === "undefined"
    ? "Or continue with email"
    : "Or continue with Email";
```

So even if the title mismatch were absent, the divider text could also trigger hydration drift.

The regression was introduced by commit `9be250f3` titled `Polish sign-in copy and labels` on 2026-05-01. The incident occurred on 2026-05-04, after that change.

**Conclusion**

This is not caused by i18n, auth state, locale loading, or browser extensions. The root cause is render-time use of `typeof window` to choose visible text, producing different initial HTML on the server and client.

**Recommended fix**

Make the initial render deterministic: use the same literal text on both server and client, for example always `Sign in` or always `Login`. If browser-only behavior is genuinely needed, defer it until after hydration with state updated from `useEffect`, but visible static copy should not depend on `typeof window`.

No code changes were made.
