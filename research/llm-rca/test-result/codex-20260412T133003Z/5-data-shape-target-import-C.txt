**Why it fails**

`parseEmployeesValue()` assumes the CSV `employees` cell always contains JSON with both `min` and `max`, for example `{"min":11,"max":50}`. In real imports, that field is optional and can plausibly be:

- empty
- a plain number like `150`
- a range string like `11-50`
- some other non-object value

When `JSON.parse(...)` returns a value that is not an object with both properties, `parsed.min` or `parsed.max` becomes `undefined`, and the next call to `.toString()` throws exactly the observed error:

`TypeError: Cannot read properties of undefined (reading 'toString')`

**Contributing Factors**

- The `employees` field is optional in the import modal and schema, but the server now treats it as if it always has a structured legacy format.
- The modal preview in [`components/modals/ImportTargetsModal.tsx`](/home/user/nextcrm-app-research/components/modals/ImportTargetsModal.tsx:140) validates only `last_name` / `company`; it does not validate or normalize `employees`.

**Conclusion**

Only use the provided CRM field names as values. Return only valid JSON, no explanation.",

            content: `CSV headers: ${JSON.stringify(headers)}\n\nCRM fields: ${JSON.stringify(TARGET_FIELDS)}\n\nReturn a JSON object like: { "CSV Header": "crm_field" or null }`,

        temperature: 0,
      });
