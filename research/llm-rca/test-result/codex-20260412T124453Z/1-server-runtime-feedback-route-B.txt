A request payload contract mismatch introduced in commit `3e3a3e1` (`Refine feedback submission flow`) is the most likely cause of the `POST /api/feedback` 500.

The client now sends:

- `feedback`
- `page`
- `submittedAt`

from [FeedbackForm.tsx](/home/user/nextcrm-app-research/app/[locale]/(routes)/components/FeedbackForm.tsx:45)

But the API route now assumes this shape:

- `body.context.pathname`

at [route.ts](/home/user/nextcrm-app-research/app/api/feedback/route.ts:34)

Because the client does not send `context`, `body.context` is `undefined`, so this line throws before the route reaches its `try/catch` around `resend.emails.send(...)`:

```ts

```

That produces an unhandled server exception and therefore a 500 response, which matches the incident symptoms.

- The error is a server-side 500, not a client validation issue.
- The failing access happens outside the route’s guarded email-send block, so it would surface as a raw handler failure.

- A repo-wide search shows no other producer of `body.context.pathname`; the only current sender uses `page` instead.
