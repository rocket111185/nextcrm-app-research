Local evidence shows `.env.local` has `RESEND_API_KEY` present but empty. If no database fallback key exists, the route returns HTTP 500 immediately with “Resend API key is not configured,” matching the Axios `Request failed with status code 500` seen in the incident.

**Secondary Defect**

Even if Resend is configured, the same endpoint likely still fails for the current client payload. The client sends:

`feedback`, `page`, `submittedAt`

But the API reads:

`body.context.pathname`

at [route.ts](/home/dmytro/Github/phd-work/nextcrm-app-update/app/api/feedback/route.ts:36).

Since `context` is not sent by the form, this would throw a `TypeError` before the mail-send `try/catch`, also producing a 500.

**Conclusion**

The most likely immediate root cause in this local incident is missing Resend configuration. There is also a deterministic request contract mismatch in the feedback endpoint that can produce the same 500 once Resend is configured. No code changes were made.
