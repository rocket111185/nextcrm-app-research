**Incident Report**

**Most likely root cause:**  
`LoginComponent` renders different text on the server and client during the initial render.

In [LoginComponent.tsx](/home/dmytro/Github/phd-work/nextcrm-app-update/app/[locale]/(auth)/sign-in/components/LoginComponent.tsx:36):

```ts
const cardTitle = typeof window === "undefined" ? "Login" : "Sign in";
```

During SSR, `typeof window === "undefined"` is true, so the server HTML contains:

```text
Login
```

During client hydration, `window` exists, so React expects:

```text
Sign in
```

That exactly matches the incident trace:

```text
+ Sign in
- Login
```

**Additional related issue:**  
The same server/client render mismatch exists for the divider label:

```ts
const dividerLabel =
  typeof window === "undefined"
    ? "Or continue with email"
    : "Or continue with Email";
```

This may also trigger hydration mismatch once the title mismatch is removed.
